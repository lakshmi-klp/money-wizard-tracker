I want to expand this application by introducing an Admin Login and Admin Dashboard system that allows the administrator to monitor application usage, user activity, and system analytics.

The admin panel should be secure and accessible only to authorized administrators.

ADMIN SYSTEM REQUIREMENTS

Please implement the following features step by step.

ADMIN USER ROLE

Modify the existing User model to include a new field:

is_admin (Boolean)

This field determines whether a user is an administrator.

Example:

is_admin = db.Column(db.Boolean, default=False)

Regular users should have is_admin = False, while administrators will have is_admin = True.

The admin account can be manually created in the database or seeded automatically when the app starts.

ADMIN LOGIN PAGE

Create a separate admin login system.

Admin login should be available at:

/admin/login

The admin login page should include:

• email input
• password input
• login button

It should not use the normal user login page.

When the admin logs in successfully:

• verify credentials
• check is_admin == True
• redirect to admin dashboard

If a normal user attempts to login through the admin page, show an error message.

ADMIN DASHBOARD

Create a new page:

/admin/dashboard

Only accessible if the logged-in user is an administrator.

This dashboard should show important system metrics including:

TOTAL REGISTERED USERS
Display the number of users registered in the system.

ACTIVE USERS
Track users who have logged in within the last 7 days.

TOTAL EXPENSE RECORDS
Count all expenses stored in the database.

TOTAL REPORT DOWNLOADS
Track how many times users download financial reports.

NEW USERS THIS MONTH
Display number of users registered during the current month.

USER ANALYTICS TABLE

Display a table on the admin dashboard showing:

User ID

Last Login Date
Total Expenses Recorded
Total Reports Downloaded

The table should allow sorting by:

• newest users

DOWNLOAD TRACKING SYSTEM

Add a new field to track report downloads.

Option 1:
Add a column in the User model:

reports_downloaded = db.Column(db.Integer, default=0)

Every time a user downloads a report, increment this counter.

Option 2:
Create a new model called ReportDownload:

ReportDownload

id

user_id

timestamp

Then count downloads per user.

Either approach is acceptable, but the system must allow the admin dashboard to display download analytics.

USER ACTIVITY TRACKING

Add a new column to track user activity:

last_login = db.Column(db.DateTime)

Update this field whenever the user logs in successfully.

Use this field to calculate:

Active Users = users who logged in during last 7 days.
